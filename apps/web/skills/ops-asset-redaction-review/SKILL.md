# 运营工作流资产投稿脱敏审核清单

## 使用场景

当同事把工具、SOP、Skill、脚本、提示词包或复盘卡点提交到待审核投稿池时，用这个流程辅助审核。

## 审核步骤

1. 判断资产是否解决了明确岗位痛点。
2. 检查输入材料、使用步骤、输出结果和失败处理是否清楚。
3. 检查人工审核边界是否明确。
4. 检查是否包含账号、密码、token、客户信息、公司私有路径或未授权链接。
5. 给出结论：通过、需补充、不建议发布。

## 输出格式

```text
审核结论：
适用岗位：
解决痛点：
可复用原因：
需补充内容：
敏感信息风险：
人工审核边界：
```

## 边界

Codex 只给审核建议。最终发布、退回或拒绝必须由管理员确认。
