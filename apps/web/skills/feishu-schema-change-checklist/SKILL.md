# Feishu 多维表格字段变更前检查与回滚记录

## 使用场景

当运营自动化流程依赖飞书字段、状态选项或表格结构，而你准备改字段名、字段类型或选项值时使用。

## 检查步骤

1. 列出拟变更字段名、字段类型、选项值和变更原因。
2. 搜索脚本、文档、测试和 SOP 中依赖该字段的位置。
3. 生成变更前检查清单、变更后验证清单和回滚说明。
4. 标出需要负责人确认的事项。

## 输出格式

```text
拟变更字段：
变更原因：
影响范围：
变更前检查：
变更后验证：
回滚方式：
负责人确认项：
```

## 边界

Codex 可以做影响分析和检查清单，但不能自行修改线上飞书表结构。
