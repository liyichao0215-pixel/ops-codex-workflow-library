# 给朋友的领取提示词

## 你需要从 GitHub 拿什么

公开仓库：

`https://github.com/liyichao0215-pixel/ops-codex-workflow-library`

人看的说明在：

`docs/Flova教程视频协同包`

这个包里有三类东西：

- `SOP.md`：给人看的教程视频协作流程。
- `skills/flova-tutorial-video-coordination`：给 Codex 用的 Skill 主版本。
- `skills/flova-tutorial-video-coordination/references/问题诊断表.md`：常见问题的诊断、修复和验收表。

仓库是公开的，不需要申请私人仓库权限。

## 怎么安装 Skill

把下面这个文件夹放到你的 Codex Skills 目录：

`skills/flova-tutorial-video-coordination`

放好后，重启或刷新 Codex，让它能识别这个 Skill。

## 第一次使用时直接发给 Codex 的提示词

```text
请使用 $flova-tutorial-video-coordination 帮我制作或质检一个教程视频。

先不要直接进入动效制作，请按照这个顺序工作：
1. 盘点我的项目目录和素材。
2. 找到最终口播稿、口播音频、BGM、截图、录屏、生成视频和当前成片。
3. 以真实口播音频作为唯一主时间轴，先校准字幕。
4. 建立“口播句子 -> 画面证据 -> 素材文件 -> 时间段”的素材证据表。
5. 先给我一个粗剪逻辑预案，说明每段口播对应什么画面。
6. 如果有横图、竖图、录屏混在一起，请按素材原始比例设计展示方式，不要拉伸。
7. 只有声音、字幕、画面证据三者匹配后，再进入 Remotion / HyperFrames 动效包装。
8. 每次返修必须带时间码，先做局部预览，通过后再渲染全片。

我的项目目标是：
【这里填写视频主题和受众】

我的素材目录是：
【这里填写本地路径】

我最担心的问题是：
【比如字幕不同步、画面和口播不匹配、素材横竖比例混乱、某一段太单一】
```

## 一句话原则

声音定节奏，字幕定理解，素材定可信度，动效定爽感；每次返修只改有时间码的问题，局部通过后再渲染全片。
