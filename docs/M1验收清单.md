# M1 验收清单

## 功能验收

- 同事能打开前端网站。
- 前端能连接云后端。
- 同事能提交投稿。
- 投稿先进入待审核池。
- 其他同事能看到待审核投稿。
- 可以登记 Codex 审核结论。
- 管理员能用 `ADMIN_TOKEN` 发布正式资产。
- 发布前投稿不会出现在正式资产列表。
- 发布后资产能出现在正式资产列表。

## 部署验收

- GitHub 仓库已创建并推送。
- Render Blueprint 能识别 `render.yaml`。
- Render 服务配置了 `ADMIN_TOKEN`。
- Render 服务配置了持久盘。
- 云端 `/api/health` 返回 `ok: true`。
- 云端页面显示“后端已连接”。

## 数据验收

- SQLite 使用持久磁盘。
- 服务重启后投稿仍在。
- 每天有备份方案。
- 可以导出正式资产。

## 安全验收

- 管理员密码不写进前端。
- 后端限制允许访问的前端域名。
- 页面提醒不要提交账号、密码、token、客户信息和公司私有路径。
- 敏感信息投稿不能直接发布。

## 体验验收

- 第一屏能看懂这是做什么。
- 三个 Codex 入口能复制可用指令。
- 投稿字段不过度复杂。
- 按钮点击后有反馈。
- Codex 指令会提醒人工审核边界。
