# 运营 Codex 工作流资产库前端工作台改版设计

## 目标

把当前说明书式页面改成运营同事能直接使用的内部工作台。第一屏必须回答三件事：我现在能做什么、资产库里有什么、投稿和审核是否正常。

## 视觉方向

- 学 Linear 的清晰工作流和低噪音信息层级。
- 学 Raycast Store 的资产库浏览、搜索和复用入口。
- 学 Dub 的指标卡和状态面板，但不做营销页风格。
- 保持内部工具气质：克制、清楚、密度适中，不用大面积渐变和装饰图形。

## 页面结构

1. 左侧导航：品牌、当前状态、核心区块锚点。
2. 顶部工作台：一句话说明、云端状态、正式资产数、待审核数、同事内测任务。
3. 指令中心：上传沉淀、找解决办法、交叉验证三个高频动作。
4. 投稿与审核流：提交表单、待审核池、刷新和发布入口。
5. 资产库：搜索、分类筛选、资产卡片。
6. 管理员设置：默认收起，只给管理员处理后端地址。

## 交互边界

- 保留现有元素 id，确保前端脚本继续绑定原来的按钮和输入框。
- 普通同事不需要看到复杂连接说明。
- 管理员功能不删除，但从主路径降噪。
- 移动端变成单列，左侧导航收成顶部状态区。

## 验收

- 页面含工作台版标记 `data-ui-version="workbench-v2"`。
- 页面显示“今日工作台”“指令中心”“投稿与审核流”“资产库”。
- 管理员设置使用可收起的 `details` 区块。
- 原有云端 smoke test 和 API 测试继续通过。
- 浏览器页面显示后端已连接、正式资产 10 条，移动端无明显溢出。
