# 部署说明

## 目标

第一阶段部署成内部内测版：同事打开前端 URL 可以提交投稿，后端保存到 SQLite，管理员审核通过后发布为正式资产。

## 推荐架构

```text
静态前端：GitHub Pages 或 Vercel
后端 API：Render / Railway / Fly.io
数据库：SQLite + 持久磁盘
```

## 后端环境变量

```text
PORT=8787
ADMIN_TOKEN=只给管理员使用的审核密码
PUBLIC_ORIGINS=https://你的前端域名
OPS_DB_PATH=/data/ops-assets.sqlite
```

## 后端健康检查

```text
GET /api/health
```

成功时返回：

```json
{
  "ok": true,
  "mode": "realtime-backend"
}
```

## 前端接入云后端

云后端地址确定后，修改 `apps/web/index.html`：

```js
window.OPS_CONFIG = {
  apiBase: 'https://你的后端服务地址'
};
```

本地访问 `127.0.0.1` 或 `localhost` 时，前端仍默认连接 `http://127.0.0.1:8787`。

## 数据持久化

云平台必须配置持久磁盘，并把 `OPS_DB_PATH` 指向持久磁盘路径。不要只使用临时容器目录。

## 备份

最低要求：

- 发布内测前手动备份一次。
- 每天自动备份一次 SQLite 文件。
- 每次管理员批量发布前导出正式资产。

本地备份命令：

```bash
node scripts/backup-sqlite.mjs
```

本地导出命令：

```bash
node scripts/export-assets.mjs
```
