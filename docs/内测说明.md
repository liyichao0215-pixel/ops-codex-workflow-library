# 内测说明

## 内测目标

让 2-3 位运营同事真实使用“运营部 Codex 工作流资产库”，验证能不能提交、审核、发布和复用工作流资产。

## 每位同事要完成的动作

1. 打开前端网站。
2. 提交 1 条自己真实用过的工具、SOP、Skill、提示词包、自动化流程或复盘卡点。
3. 用“找岗位痛点解决办法”入口，让自己的 Codex 搜索 1 个真实痛点。
4. 审核 1 条别人的投稿，登记审核结论。

## 管理员要完成的动作

1. 检查待审核投稿是否包含账号、密码、token、客户信息、公司私有路径或未授权链接。
2. 确认至少有团队 Codex 审核结论。
3. 对可发布投稿点击“审核通过发布”。
4. 记录同事反馈和页面卡点。

## 成功标准

- 至少 3 位同事参与。
- 至少 3 条投稿进入待审核池。
- 至少 1 条投稿发布为正式资产。
- 同事能在 3 分钟内从岗位痛点找到可复用方案。
- 敏感信息误发布为 0。
