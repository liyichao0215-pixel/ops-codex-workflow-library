# 内容归属与唯一真源

## 仓库定位

这是公开的、已脱敏的运营工作流发行仓库。它负责让别人能够查看、安装、测试和复用通用资产，不负责保存个人生产现场。

## 目录职责

| 目录 | 职责 | 是否可直接编辑 |
|---|---|---|
| `skills/` | 可安装 Skill 主版本 | 是 |
| `docs/` | SOP、领取说明、审核与部署文档 | 是 |
| `apps/api/` | 投稿、审核和资产服务 | 是 |
| `apps/web/` | 静态前端和发布镜像 | 代码可编辑；`apps/web/skills/` 不单独编辑 |
| `data/` | 可公开的种子资产定义 | 是，但必须脱敏 |
| `scripts/` | 构建、导出、备份和一致性检查 | 是 |

## 不进入本仓库

- 私人账号表达记忆、账号清单和实时排期。
- Token、Cookie、邮箱、手机号、客户信息和未授权链接。
- 本机绝对路径、浏览器 profile、运行日志和 Postiz readback。
- 原始视频、竞品视频、截图、录屏、音频和一次性渲染产物。
- 只能在私人 Flova 项目或登录态中使用的内容。

## 重复文件规则

`apps/web/skills/` 是为了静态网站直接访问而保留的发行镜像。根目录 `skills/` 才是唯一主版本。修改流程：

1. 只修改 `skills/<name>/SKILL.md`。
2. 需要网页发行时再同步到 `apps/web/skills/<name>/SKILL.md`。
3. 运行测试；镜像内容不同会直接失败。

教程协同包同理：`docs/Flova教程视频协同包/` 保存人类说明，`skills/flova-tutorial-video-coordination/` 保存可安装 Skill，不在 Obsidian 或私人教程仓库维护第二份。
